# Clarification Questions

Ask before creating a task when one wrong assumption would make the reminder annoying or useless. Keep questions short and ask only what blocks a safe task.

## Timing Is Missing

Use when the user says "提醒我" but gives no time.

- "你希望我什么时候提醒？"
- "这是今天提醒，还是每天/每周都要提醒？"
- "如果只是尽快提醒一次，我可以现在创建一个 asap 任务，可以吗？"

## Recurrence Is Ambiguous

Use when the user gives a habit or repeated action but not the cadence.

- "这个提醒要每隔多久一次？"
- "是每天固定时间，还是每完成一次后重新计时？"
- "循环从什么时候开始？"

## Completion Anchor Is Ambiguous

Use especially for drinking water, standing up, stretching, eye rest, medicine, and chores.

- "下一次提醒要按固定时间算，还是从你确认完成后重新计时？"
- "如果你晚一点完成，下次提醒也顺延吗？"
- "这个任务需要你确认完成吗？"

## Follow-Up Is Ambiguous

Use when the user asks to be reminded about something important but does not say what to do if they miss it.

- "如果你没有确认完成，我要追问几次？"
- "追问间隔希望是多少？"
- "追问到一定次数后，要不要询问是否改期？"

## Deadline Versus Exact Reminder

Use when the user says "之前" or "截止" but might mean an early reminder.

- "你希望在截止时间提醒，还是提前一段时间提醒？"
- "这个任务可以提前完成吗？"
- "到截止时间还没完成时，要继续追问吗？"

## Floating Versus Scheduled

Use when the user says "有空", "找时间", "以后", or "某天".

- "这是先放进待规划池，还是现在就定一个提醒时间？"
- "有没有最晚完成时间？"
- "你希望我主动追问你安排时间吗？"

## Executor Delivery

Use when it is unclear whether reminders should go through local desktop notification or an agent channel.

- "这个提醒要走本地通知，还是通过 OpenClaw/Hermes 发到聊天渠道？"
- "如果走 agent 渠道，要使用哪个 executor？"
